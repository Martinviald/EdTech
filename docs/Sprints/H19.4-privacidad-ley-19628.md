# Plan de Implementación — H19.4: Privacidad y Cumplimiento Ley 19.628

> **Sprint:** S0 Cimientos · **Prioridad:** P0 Bloqueante Legal · **Estimación:** 8 SP
> **Estado:** Pendiente de implementación

## Contexto

H19.4 es un bloqueante legal que debe implementarse antes de que cualquier endpoint con datos de alumnos salga a producción. Actualmente el API (NestJS) no tiene autenticación, guards ni ninguna capa de autorización. La BD no tiene políticas RLS.

Esta historia implementa cuatro capas de defensa:

1. **Autenticación JWT** en el API — Guard global que valida el token de NextAuth
2. **Autorización por rol** — Guards granulares para roles y datos sensibles
3. **RLS en PostgreSQL** — Segunda barrera a nivel de motor de base de datos
4. **Anonimización PII + auditoría** — Derecho al Olvido y trazabilidad de exportaciones

## Step 2 — `withOrgContext()`: wrapper transaccional para RLS

### `packages/db/src/with-org-context.ts` — CREAR

```typescript
import { sql } from 'drizzle-orm';
import type { Database } from './client';

/**
 * Envuelve una operación en una transacción que fija app.current_org_id
 * para las políticas RLS. El SET LOCAL es transaction-scoped: se resetea
 * automáticamente al terminar la transacción (seguro con connection pooling).
 */
export async function withOrgContext<T>(
  db: Database,
  orgId: string,
  fn: (tx: Database) => Promise<T>,
): Promise<T> {
  return db.transaction(async (tx) => {
    await tx.execute(sql`SELECT set_config('app.current_org_id', ${orgId}, true)`);
    return fn(tx as unknown as Database);
  });
}
```

### `apps/api/src/auth/auth.guard.ts` — CREAR

Valida el JWE (token de NextAuth v5) usando `jose`. NextAuth v5 cifra los JWT con AES-256-CBC usando `AUTH_SECRET`.

```typescript
import { CanActivate, ExecutionContext, Injectable, UnauthorizedException } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import { Reflector } from '@nestjs/core';
import { jwtDecrypt } from 'jose';
import { IS_PUBLIC_KEY } from '../common/decorators/public.decorator';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(
    private readonly config: ConfigService,
    private readonly reflector: Reflector,
  ) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);
    if (isPublic) return true;

    const request = context.switchToHttp().getRequest();
    const token = this.extractToken(request);
    if (!token) throw new UnauthorizedException('Token requerido');

    try {
      const secret = new TextEncoder().encode(this.config.getOrThrow<string>('AUTH_SECRET'));
      const { payload } = await jwtDecrypt(token, secret);

      request.user = {
        userId: payload['userId'] as string,
        orgId: payload['orgId'] as string,
        role: payload['role'] as string,
        email: payload['email'] as string,
        name: payload['name'] as string,
      };
      return true;
    } catch {
      throw new UnauthorizedException('Token inválido o expirado');
    }
  }

  private extractToken(request: Request): string | undefined {
    const authHeader = (request.headers as Record<string, string>)['authorization'];
    const [type, token] = authHeader?.split(' ') ?? [];
    return type === 'Bearer' ? token : undefined;
  }
}
```

```typescript
import { CanActivate, ExecutionContext, ForbiddenException, Injectable } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { ROLES_KEY } from '../decorators/roles.decorator';
import type { JwtPayload } from '../../auth/jwt-payload.types';

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private readonly reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const required = this.reflector.getAllAndOverride<string[]>(ROLES_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);
    if (!required?.length) return true;

    const { user } = context.switchToHttp().getRequest() as { user: JwtPayload };
    if (!required.includes(user.role)) {
      throw new ForbiddenException('Rol insuficiente para esta operación');
    }
    return true;
  }
}
```

## Step 5 — `PrivacyService`: anonimización y auditoría

### `apps/api/src/privacy/privacy.service.ts` — CREAR

```typescript
import { Injectable, NotFoundException } from '@nestjs/common';
import { createHash } from 'crypto';
import { eq, isNull } from 'drizzle-orm';
import { InjectDb, type Database } from '../database/database.types';
import { students, auditLogs } from '@edtech/db';

@Injectable()
export class PrivacyService {
  constructor(@InjectDb() private readonly db: Database) {}

  // ── Derecho al Olvido (Art. 12, Ley 19.628) ──────────────────────

  async anonymizeStudent(
    studentId: string,
    requestedBy: { userId: string; orgId: string },
  ): Promise<void> {
    const [student] = await this.db
      .select({ id: students.id, orgId: students.orgId, isAnonymized: students.isAnonymized })
      .from(students)
      .where(eq(students.id, studentId));

    if (!student || student.orgId !== requestedBy.orgId) {
      throw new NotFoundException('Alumno no encontrado');
    }
    if (student.isAnonymized) return; // operación idempotente

    // SHA-256 sobre el UUID del alumno (no sobre el RUT real — no reversible)
    const anon = (salt: string) =>
      'anon-' +
      createHash('sha256')
        .update(studentId + salt)
        .digest('hex')
        .slice(0, 16);

    await this.db.transaction(async (tx) => {
      await tx
        .update(students)
        .set({
          rut: anon('rut'),
          firstName: 'Anon',
          lastName: anon('ln'),
          birthDate: null, // PII
          profile: {}, // elimina NEE, sensitiveNotes, etc.
          userId: null, // desvincula cuenta digital
          isAnonymized: true,
          updatedAt: new Date(),
        })
        .where(eq(students.id, studentId));

      await tx.insert(auditLogs).values({
        userId: requestedBy.userId,
        orgId: requestedBy.orgId,
        action: 'anonymize_student',
        resourceType: 'students',
        resourceFilter: { studentId },
        recordCount: 1,
      });
    });
    // Los datos estadísticos (responses, assessment_results, skill_results)
    // se conservan intactos — son anónimos por diseño al eliminar el PII del alumno.
  }

  // ── Auditoría de exportaciones masivas ──────────────────────────

  async logExport(params: {
    userId: string;
    orgId: string;
    action: string;
    resourceType: string;
    resourceFilter?: Record<string, unknown>;
    recordCount?: number;
    ipAddress?: string;
    userAgent?: string;
  }): Promise<void> {
    await this.db.insert(auditLogs).values(params);
  }

  async listAuditLogs(orgId: string, limit = 100) {
    return this.db
      .select()
      .from(auditLogs)
      .where(eq(auditLogs.orgId, orgId))
      .orderBy(auditLogs.createdAt)
      .limit(limit);
  }
}
```

```typescript
import { Controller, Post, Get, Param, Query, UseGuards } from '@nestjs/common';
import { PrivacyService } from './privacy.service';
import { Roles } from '../common/decorators/roles.decorator';
import { RolesGuard } from '../common/guards/roles.guard';
import { CurrentUser } from '../auth/current-user.decorator';
import type { JwtPayload } from '../auth/jwt-payload.types';

@Controller('privacy')
@UseGuards(RolesGuard)
export class PrivacyController {
  constructor(private readonly privacyService: PrivacyService) {}

  // POST /api/privacy/students/:id/anonymize
  @Post('students/:id/anonymize')
  @Roles('school_admin', 'platform_admin')
  async anonymizeStudent(@Param('id') studentId: string, @CurrentUser() user: JwtPayload) {
    await this.privacyService.anonymizeStudent(studentId, {
      userId: user.userId,
      orgId: user.orgId,
    });
    return { success: true, message: 'Datos del alumno anonimizados correctamente' };
  }

  // GET /api/privacy/audit-logs
  @Get('audit-logs')
  @Roles('school_admin', 'academic_director', 'platform_admin', 'foundation_director')
  async getAuditLogs(@CurrentUser() user: JwtPayload, @Query('limit') limit?: string) {
    return this.privacyService.listAuditLogs(user.orgId, limit ? parseInt(limit) : 100);
  }
}
```

## Step 7 — Políticas RLS en PostgreSQL

Agregar manualmente este bloque al final del archivo `.sql` generado en Step 1 **antes** de aplicar la migración.

```sql
-- ================================================================
-- H19.4 — Row Level Security (aislamiento multi-tenant Ley 19.628)
-- ================================================================

-- Tablas con org_id directo
ALTER TABLE students        ENABLE ROW LEVEL SECURITY;
ALTER TABLE assessments     ENABLE ROW LEVEL SECURITY;
ALTER TABLE import_jobs     ENABLE ROW LEVEL SECURITY;

-- Tablas sin org_id directo (heredan vía assessments)
ALTER TABLE responses           ENABLE ROW LEVEL SECURITY;
ALTER TABLE assessment_results  ENABLE ROW LEVEL SECURITY;
ALTER TABLE skill_results       ENABLE ROW LEVEL SECURITY;

-- ── Políticas ──────────────────────────────────────────────────
-- current_setting(..., true) retorna '' si no está definido (no lanza error).
-- Si la variable no está fijada, la condición es falsa → 0 filas devueltas (safe default).

CREATE POLICY students_tenant_isolation ON students
  AS PERMISSIVE FOR ALL
  USING (org_id::text = current_setting('app.current_org_id', true));

CREATE POLICY assessments_tenant_isolation ON assessments
  AS PERMISSIVE FOR ALL
  USING (org_id::text = current_setting('app.current_org_id', true));

CREATE POLICY import_jobs_tenant_isolation ON import_jobs
  AS PERMISSIVE FOR ALL
  USING (org_id::text = current_setting('app.current_org_id', true));

-- responses: sin org_id → subquery a assessments
CREATE POLICY responses_tenant_isolation ON responses
  AS PERMISSIVE FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM assessments
      WHERE id = responses.assessment_id
        AND org_id::text = current_setting('app.current_org_id', true)
    )
  );

-- assessment_results: sin org_id → subquery a assessments
CREATE POLICY assessment_results_tenant_isolation ON assessment_results
  AS PERMISSIVE FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM assessments
      WHERE id = assessment_results.assessment_id
        AND org_id::text = current_setting('app.current_org_id', true)
    )
  );

-- skill_results: sin org_id → subquery a assessments
CREATE POLICY skill_results_tenant_isolation ON skill_results
  AS PERMISSIVE FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM assessments
      WHERE id = skill_results.assessment_id
        AND org_id::text = current_setting('app.current_org_id', true)
    )
  );

-- ================================================================
-- ADVERTENCIA: El usuario de BD usado por el API (soe_app) NO debe
-- tener BYPASSRLS. Ver sección "Fuera de scope" para configuración.
-- ================================================================
```

## Patrón de uso para futuros services

Todo Service que consulte tablas con RLS activo debe usar `withOrgContext`:

```typescript
import { withOrgContext } from '@edtech/db';

@Injectable()
export class StudentsService {
  constructor(@InjectDb() private readonly db: Database) {}

  // Sin datos sensibles — para profesores y roles estándar
  async findAll(orgId: string) {
    return withOrgContext(this.db, orgId, (tx) =>
      tx
        .select({
          id: students.id,
          rut: students.rut,
          firstName: students.firstName,
          lastName: students.lastName,
          isAnonymized: students.isAnonymized,
          // profile excluido intencionalmente
        })
        .from(students)
        .where(isNull(students.deletedAt)),
    );
  }

  // Con datos sensibles — solo para roles autorizados (protegido con SensitiveDataGuard)
  async findOneSensitive(id: string, orgId: string) {
    const [student] = await withOrgContext(this.db, orgId, (tx) =>
      tx.select().from(students).where(eq(students.id, id)),
    );
    return student;
  }
}
```

## Fuera de scope — Requiere infra, legal o proceso externo

### BD — Roles de PostgreSQL (configuración de infra)

Para que el RLS funcione correctamente con la app y no bloquee migraciones, se necesitan **dos usuarios de BD**:

```sql
-- Usuario de la app (sujeto a RLS — usado en DATABASE_URL de producción)
CREATE ROLE soe_app LOGIN PASSWORD '...' NOSUPERUSER NOINHERIT NOCREATEDB;
GRANT CONNECT ON DATABASE soe TO soe_app;

-- Usuario de migraciones (BYPASSRLS — usado solo en pnpm db:migrate)
CREATE ROLE soe_admin LOGIN PASSWORD '...' NOSUPERUSER BYPASSRLS CREATEDB;
```

En desarrollo con un solo usuario, deshabilitar RLS temporalmente para migraciones:

