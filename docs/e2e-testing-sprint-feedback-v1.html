Antes de empezar — requisitos, setup y arranque §1–3

1 · Requisitos

  • PostgreSQL local con la base soe_dev y dos roles: soe_app sin BYPASSRLS (lo usa la API → DATABASE_URL) y un rol privilegiado para migrate/seed (DATABASE_ADMIN_URL).
  • Node + pnpm 10.x.
  • .env en la raíz (base: apps/api/.env.example):
DATABASE_URL=postgresql://soe_app:...@localhost:5432/soe_dev
DATABASE_ADMIN_URL=postgresql://<admin>:...@localhost:5432/soe_dev
AUTH_MODE=mock            # login local sin SSO
AUTH_SECRET=cualquier-string
NEXTAUTH_SECRET=cualquier-string
NEXTAUTH_URL=http://localhost:3000
API_PORT=4000
API_URL=http://localhost:4000
INTERNAL_API_SECRET=cualquier-string
  • Opcionales por feature: GEMINI_API_KEY → TKT-19 y TKT-23. Config S3 (STORAGE_S3_BUCKET, AWS_*) → TKT-15 (sin ella, el endpoint responde 503 y el resto funciona).

2 · Setup

pnpm install
pnpm --filter @soe/types build && pnpm --filter @soe/db build
pnpm --filter @soe/db db:migrate     # aplica 0008 + 0009 y re-aplica RLS
pnpm --filter @soe/db db:seed:dev    # org demo + taxonomía + instrumentos + tags + resultados

db:migrate y db:seed:dev usan DATABASE_ADMIN_URL.

3 · Levantar y loguear

pnpm --filter @soe/api dev    # → http://localhost:4000
pnpm --filter @soe/web dev    # → http://localhost:3000

Abre localhost:3000/login y elige un usuario mock: admin.demo@colegiodemo.cl (directivo, ve toda la org) para la mayoría; profesor.demo@colegiodemo.cl (profesor, solo sus cursos) para verificar scoping/RLS.
